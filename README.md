Sprint 3 – Urban Routes (Teste Manual Completo)
Descrição do projeto

- Realização da validação completa do sistema Urban Routes, incluindo checklist de layout, método de pagamento, cadastro de cartão, botão "Reservar", fluxo de aluguel e registro de bugs no Jira.

Objetivo

- Garantir que as funcionalidades críticas da aplicação funcionassem corretamente antes da publicação.

Resultado

- Foram produzidos checklists completos, casos de teste detalhados e diversos relatórios de bugs registrados no Jira.

Ferramentas utilizadas
- Jira
  
- Google Sheets
  
- Chrome
  
- DevTools
  
O que aprendi com esse projeto (habilidades/competências adquiridas)

- Organização de testes
  
- Uso profissional do Jira
  
- Priorização de defeitos
  
- Criação de checklists
  
- Cobertura de testes

- Fluxo completo de QA Manual
  
Melhorias possíveis

- Automatizar o fluxo de reserva.
  
- Implementar testes de regressão automatizados.
  
- Metodologia


Teste manual baseado em requisitos funcionais e critérios de aceitação.

Como executar

- Abrir aplicação.
  
- Seguir o checklist.
  
- Executar casos de teste.
  
- Registrar defeitos encontrados.
  
- Validar correções.
